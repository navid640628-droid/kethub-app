# کتاب‌باز — پروژه‌ی PWA کتاب، کتاب صوتی و پادکست

پروژه‌ای سبک و بدون بک‌اند، فقط با HTML + CSS + JavaScript خام، برای انتشار روزانه‌ی محتوای کتاب.

## ساختار پوشه‌ها

```
/
├── index.html              صفحه‌ی اصلی (لیست کتاب‌ها، فیلتر، جست‌وجو، پادکست‌ها)
├── manifest.json           تنظیمات PWA
├── sw.js                   سرویس ورکر (کش آفلاین)
├── robots.txt              راهنمای خزنده‌های گوگل
├── sitemap.xml             نقشه‌ی سایت (با generate-sitemap.py به‌روز می‌شود)
├── generate-sitemap.py     اسکریپت به‌روزرسانی sitemap.xml (اجرای محلی)
├── assets/
│   ├── images/
│   ├── icons/              آیکون‌های اپ (icon-192.svg, icon-512.svg)
│   └── covers/             جلد کتاب‌ها، پادکست‌ها و عکس مشاهیر (در پوشه‌ی figures/)
├── data/
│   ├── books.json          ← برای افزودن کتاب
│   ├── podcasts.json       ← برای افزودن قسمت پادکست
│   ├── marginalia.json     ← برای افزودن حاشیه/حقیقت خواندنی
│   ├── figures.json        ← برای افزودن چهره‌ی جدید به مشاهیر
│   └── categories.json     ژانرها، نوع محتوا، زبان، امتیاز
├── js/
│   ├── app.js               راه‌انداز صفحه‌ی اصلی
│   ├── books.js             واکشی/فیلتر/رندر کتاب‌ها
│   ├── figures.js           واکشی/رندر مشاهیر
│   ├── audio-player.js       پلیر صوتی اختصاصی (کتاب صوتی + پادکست)
│   ├── bottom-nav.js         نوار ناوبری ثابت پایین صفحه (موبایل)
│   ├── storage.js            علاقه‌مندی، شناسه‌ی کاربر، موقعیت پخش
│   └── seo.js                متا تگ‌های داینامیک، canonical و JSON-LD
├── css/
│   ├── style.css
│   └── responsive.css
└── pages/
    ├── book.html             صفحه‌ی اختصاصی هر کتاب (book.html?id=...)
    ├── category.html         صفحه‌ی دسته‌بندی (category.html?genre=novel)
    ├── podcasts.html         فهرست کامل پادکست‌ها
    ├── marginalia.html       فهرست حاشیه‌ها و حقایق خواندنی
    ├── figures.html          فهرست مشاهیر (با فیلتر دسته‌بندی)
    └── figure.html           صفحه‌ی زندگی‌نامه‌ی هر چهره (figure.html?id=...)
```

## افزودن کتاب جدید (بدون نیاز به تغییر کد)

در گیت‌هاب، فایل `data/books.json` را باز کن و یک آبجکت جدید **در ابتدای آرایه** اضافه کن:

```json
{
  "id": "shناسه-یکتا-انگلیسی",
  "title": "عنوان کتاب",
  "author": "نویسنده",
  "genre": "novel",
  "language": "fa",
  "type": "review",
  "rating": "top",
  "score": 4.5,
  "cover": "assets/covers/نام-فایل.svg",
  "audio": null,
  "pdf": null,
  "summary": "خلاصه‌ی کوتاه کتاب...",
  "highlights": ["نکته ۱", "نکته ۲", "نکته ۳"],
  "telegram": "https://t.me/kmketab",
  "tags": ["برچسب۱", "برچسب۲"]
}
```

مقادیر مجاز:
- `genre`: novel, science, self-growth, history-philosophy, poetry
- `language`: fa, translated
- `type`: review, summary, intro
- `rating`: top, special (یا حذف این فیلد برای کتاب‌های عادی)

برای افزودن کتاب صوتی، مقدار `audio` را مسیر فایل mp3 (مثلاً `assets/audio/کتاب/ch1.mp3`) قرار بده. برای PDF هم همین‌طور. `id` باید یکتا و بدون فاصله باشد (لاتین یا اسلاگ فارسی-لاتین).

جلد کتاب را هم به‌صورت svg یا jpg/png در `assets/covers/` آپلود کن.

## افزودن قسمت جدید پادکست

فایل `data/podcasts.json` را باز کن و یک آبجکت جدید در ابتدای آرایه اضافه کن (ساختار مشابه است: title, host, cover, audio, duration, description).

## افزودن حاشیه‌ی جدید (حقایق و نکات خواندنی)

فایل `data/marginalia.json` را باز کن و یک آبجکت جدید در ابتدای آرایه اضافه کن:

```json
{
  "id": "شناسه-یکتا",
  "bookId": "شناسه-کتاب-مرتبط-یا-null",
  "bookTitle": "عنوان کتاب (اگر bookId پر است)",
  "text": "متن حاشیه یا حقیقت خواندنی...",
  "tags": ["برچسب۱", "برچسب۲"]
}
```

اگر `bookId` را پر کنی، این حاشیه هم در صفحه‌ی `pages/marginalia.html` و هم به‌صورت خودکار در صفحه‌ی اختصاصی همان کتاب (`pages/book.html?id=...`) نمایش داده می‌شود. اگر حاشیه به کتاب خاصی مربوط نیست، `bookId` را `null` بگذار.

## افزودن چهره‌ی جدید به «مشاهیر»

فایل `data/figures.json` را باز کن و یک آبجکت جدید در ابتدای آرایه اضافه کن:

```json
{
  "id": "شناسه-یکتا-انگلیسی",
  "name": "نام کامل",
  "category": "writer",
  "photo": "assets/covers/figures/نام-فایل.svg",
  "birthYear": 1900,
  "deathYear": 1980,
  "nationality": "کشور",
  "shortBio": "یک جمله‌ی کوتاه معرفی (برای کارت لیست)",
  "biography": "زندگی‌نامه‌ی کامل‌تر (چند پاراگراف، برای صفحه‌ی اختصاصی)",
  "relatedBookIds": ["شناسه-کتاب-۱", "شناسه-کتاب-۲"],
  "tags": ["برچسب۱", "برچسب۲"]
}
```

مقادیر مجاز `category`: `writer` (نویسنده)، `philosopher` (فیلسوف)، `poet` (شاعر)، `scientist` (دانشمند).

عکس این چهره را در `assets/covers/figures/` آپلود کن. اگر `relatedBookIds` را پر کنی، کتاب‌های مرتبط به‌صورت خودکار پایین صفحه‌ی زندگی‌نامه نمایش داده می‌شوند.

## نوار پایین صفحه (موبایل)

فایل `js/bottom-nav.js` به‌صورت خودکار یک نوار ناوبری ثابت پایین صفحه (کتاب‌ها / پادکست / حاشیه‌ها / مشاهیر) به همه‌ی صفحات اضافه می‌کند. برای اضافه‌کردن این نوار به یک صفحه‌ی جدید، فقط کافی‌ست همین اسکریپت را در انتهای `<body>` اضافه کنی؛ نیازی به نوشتن HTML اضافه نیست. این نوار در دسکتاپ (عرض بیشتر از ۸۶۰px) به‌طور خودکار مخفی می‌شود چون در آن حالت منوی بالای صفحه کافی است.

## به‌روزرسانی نقشه‌ی سایت (Sitemap)

هر بار که کتاب، حاشیه یا چهره‌ی جدیدی اضافه کردی، فایل `generate-sitemap.py` را **روی کامپیوتر خودت** (نه روی Cloudflare) اجرا کن تا `sitemap.xml` به‌روز شود:

```
python3 generate-sitemap.py
```

بعد فایل `sitemap.xml` جدید را هم مثل بقیه‌ی فایل‌ها در گیت‌هاب کامیت کن.


## دیپلوی

پروژه هیچ مرحله‌ی build ندارد؛ کافی‌ست ریشه‌ی همین پوشه را روی Cloudflare Pages دیپلوی کنی (Build command خالی، Output directory = `/`).

## سیستم مدیریت مطالعه‌ی شخصی (Goodreads-style)

طراحی کامل معماری، ساختار پایگاه‌داده (Supabase/Postgres) و نقشه‌ی راه پیاده‌سازی سیستم کاربری و پیگیری مطالعه، در پوشه‌ی `docs/` قرار دارد:
- `docs/goodreads-system-design.md` — معماری، سطح API، نقشه‌ی راه مرحله‌به‌مرحله
- `docs/supabase-schema.sql` — کل ساختار پایگاه‌داده، آماده‌ی اجرا در Supabase SQL Editor

⚠️ این بخش هنوز پیاده‌سازی نشده و صرفاً طراحی است؛ سایت فعلی همچنان کاملاً استاتیک است.

## کتابخانه‌ی شخصی مطالعه (بدون ثبت‌نام)

صفحه‌ی `pages/my-library.html` یک فضای کاملاً شخصی برای هر بازدیدکننده است — بدون نیاز به ثبت‌نام، تمام داده در `localStorage` همان مرورگر ذخیره می‌شود (منطقش در `js/library.js` است).

**قابلیت‌ها:**
- ۴ لیست: می‌خوام بخونم / در حال خواندن / خوانده‌ام / رها شده
- برای کتاب‌های «در حال خواندن»: تعیین برنامه بر اساس ساعت آزاد روزانه و سرعت مطالعه (صفحه در ساعت) → سیستم خودکار صفحه‌ی هدف روزانه و تاریخ تخمینی پایان را حساب می‌کند و می‌گوید جلوتر یا عقب‌تر از برنامه‌ای
- ثبت روزانه‌ی تعداد صفحات خوانده‌شده
- برای کتاب‌های «خوانده‌ام»: امتیاز ۱ تا ۵ ستاره + یادداشت شخصی
- افزودن کتاب مستقیم از صفحه‌ی هر کتاب در سایت (`pages/book.html`)، یا افزودن کتاب دلخواه (خارج از کاتالوگ سایت) از خودِ صفحه‌ی کتابخانه

**نکته‌ی مهم برای Navid:** اگر فیلد اختیاری `"pages": 320` رو به هر آیتم در `data/books.json` اضافه کنی، وقتی کاربر اون کتاب رو از صفحه‌ی کتاب به کتابخانه‌اش اضافه کنه، تعداد کل صفحات خودکار پر می‌شه و نیازی نیست کاربر دستی واردش کنه.

⚠️ چون داده فقط روی مرورگر خودِ کاربره، اگه کش سایت رو پاک کنه یا از دستگاه دیگه‌ای بیاد، لیستش از اول شروع می‌شه. طراحی و مسیر ارتقا به نسخه‌ی مبتنی بر حساب کاربری (با سینک بین دستگاه‌ها) در `docs/goodreads-system-design.md` مستند شده و هر وقت خواستی می‌تونیم بهش مهاجرت کنیم.

## نکات فنی

- برای اجرای محلی باید از یک سرور استاتیک ساده استفاده کنی (مثلاً افزونه‌ی Live Server)، چون `fetch` روی `file://` در برخی مرورگرها کار نمی‌کند.
- آیکون‌های `assets/icons/` به‌صورت SVG placeholder هستند؛ برای پشتیبانی کامل‌تر از موبایل‌های قدیمی‌تر، بهتر است نسخه‌ی PNG (192×192 و 512×512) هم اضافه کنی.
